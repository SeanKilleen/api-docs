Power-Up Capabilities

There are currently 10 capabilities. Capabilities represent areas of the UX that you can hook into. For each capability you state that you support, the web application will make a request to your Power-Up for UX and functionality to be rendered within Trello, at the time when that UX item would normally be rendered.

  1. board-buttons
    • Add a single button to a board at the top of the screen
  2. card-badges
    • Allows you to add a badges to the front of cards
    • Called when opening a board.
    • Card badges are also refreshed on an interval that you specify to keep the UI fresh
  3. card-detail-badges
    • Allows you to add badges on the back of cards
    • rendered when on the back of the card
  4. attachment-sections
    • Allows you to add sections to back of a card, each containing an iFrame
    • Called when opening a card.
    • You will need to iterate through a list of Card attachments stored in options.entries
    • Return list of URLs that you are claiming
  5. attachment-thumbnail
    • Allows you to improve the attachment preview on the back of a card, for when you don't need a whole iFrame,
    • Called when opening viewing the back of a card
    • Specify the link, the text shown for “opening” the attachment, and the thumbnail
  6. card-buttons
    • Rendered when on the back of the card
    • return list of buttons { icon, text, callback -> method}
  7. format-url
    • Allows you to modify the rendering of activity on the sidebar, and any time we would display a URL
    • Rendered in the sidebar, in markdown, and anywhere else URLs are shown
    • Make URLs nicer
    • Return an update icon and text
  8. card-from-url
    • Use information that you know (via authentication or algorithmically), to handle drag and dropped URLs
    • Set the name and description of the card based on the URL being dropped
  9. show-settings
    • If you want a gear icon in the menu bar for configuring your Power-Up.
    • If you are setting board-level attributes such as which GitHub repository is on a board
