Attaching a File

Typically when attempting to attach a file to a Card, you will use the POST /1/cards/[card id or shortlink]/attachments route. Within your POST data, we expect you to send us a file.

Currently the client.js library provided by Trello does not support file uploads out of the box.

What type of file is Trello expecting?

Trello is expecting a multipart encoded file as part of a FormData object.

This generally means that the Request Payload as part of your POST will look something like this:

------WebKitFormBoundaryEeLAQ7WFZ54jDCko
Content-Disposition: form-data; name="token"

55bfd3d17144f1f19e679687/dKfxnyEha63XfdKfxnyEha63XfdKfxnyEha63XfdKfxnyEha63XfEQWKwqMiLbCG
------WebKitFormBoundaryEeLAQ7WFZ54jDCko
Content-Disposition: form-data; name="file"; filename="My-File-Name.png"
Content-Type: image/png


------WebKitFormBoundaryEeLAQ7WFZ54jDCko--
